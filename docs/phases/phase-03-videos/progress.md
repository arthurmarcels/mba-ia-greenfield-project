- **Decisions / handback notes:**
  - **`VideoNotFoundException` created here (not deferred to SI-03.6):** the service's id-keyed action methods (`completeUpload`/`markReady`/`markError`) resolve a row before acting and need a clean not-found signal → 404. SI-03.6 reuses it (same as `ForbiddenResourceException` was reassigned here because the service throws it). The finders (`findById`/`findBySlug`/`findByChannel`) return `null`/array and leave absence translation to the caller, per the repo's `findOne` rule.
  - **`IllegalVideoStatusTransitionException`** (409, `ILLEGAL_VIDEO_STATUS_TRANSITION`) thrown from the private `assertTransition` on every status change; constructor takes plain `string` statuses so `common/exceptions` stays decoupled from the videos domain.
  - **Storage stubbed in the integration spec** (DB + Redis real): `storage.service.integration-spec.ts` already proves the real MinIO multipart round-trip; this suite isolates the service's DB writes + real-Redis enqueue. The stub is kept as `jest.Mock`-typed props (cast to `StorageService` only at the DI site) so mock assertions don't trip `@typescript-eslint/unbound-method`.
  - **`completeUpload` ordering (spec):** storage-complete → persist `processing` → `queue.add`. If `queue.add` fails after the persist, the row is left `processing` with no job — accepted for SI-03.5 (`attempts:3` covers worker-side failures); outbox/enqueue-retry hardening deferred to SI-03.7+. Not a blocker.
  - **Lint baseline unchanged:** ~190 pre-existing ESLint errors in untouched files remain (per the SI DoD); only SI-03.5 files kept clean.
